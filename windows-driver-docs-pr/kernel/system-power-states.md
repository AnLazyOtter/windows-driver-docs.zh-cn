# <a name="system-power-states"></a>系统电源状态





系统电源状态描述系统的作为一个整体的功率消耗。 操作系统支持六个系统的电源状态，称为 S0 （完全打开和操作） 到 S5 （关闭电源）。 每个状态的特征如下：

-   电源消耗： 该计算机使用多少能源？

-   软件恢复： 从哪个点 does 操作系统重启？

-   硬件延迟： 它需要多长以将计算机恢复到工作状态？

-   系统硬件上下文 （例如易失性处理器寄存器、 内存缓存和 RAM 的内容）： 保留多少系统硬件上下文？ 必须在操作系统重新启动才能返回到工作状态？

状态 S0 处于工作状态。 状态 S1、 S2、 S3 和 S4 正在睡眠的状态，计算机看似已经关机由于降低的功率消耗，但会保留足够的上下文来返回到工作状态无需重新启动操作系统。 状态 S5 已关闭，或关闭状态。

系统，则*唤醒*时它正在从关闭状态 (S5) 或任何睡眠状态 (S1-S4) 转换到工作状态 (S0)，它会进入睡眠状态时从工作状态过渡到任何睡眠状态或关闭状态. 下图显示可能的系统电源状态转换。

![说明可能的系统电源状态转换的关系图](images/sysstate.png)

上图所示，系统不能直接从另一个; 输入一个睡眠状态它始终必须进入任何休眠状态前进入工作状态。 例如，系统就不能转换从状态 S2 到 S4，也不从状态 S4 到 S2。 它必须首先返回为 S0，从中，它可进入下一步的睡眠状态。 由于中间睡眠状态中的系统已丢失某些操作的上下文，它必须返回到要还原该上下文才能将其他状态转换的工作状态。
