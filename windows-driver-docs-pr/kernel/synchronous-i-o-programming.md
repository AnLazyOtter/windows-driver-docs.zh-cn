# <a name="synchronous-io-programming"></a>同步 I/O 编程


同步编程只需等待调用返回。 这是快速、 高效地从程序员的角度来看，但在像许多程序都同时运行的 Windows 环境中，它可能会导致问题。 只要可能，使用[异步 I/O 编程技术](asynchronous-i-o-programming.md)。

**请注意**  对于使用 Microsoft Vista 驱动程序开发人员，这不是同样严重的问题。 有关在 Vista 中的同步编程的详细信息，请参阅[Vista 中限制等待](restricting-waits-in-vista.md)。
