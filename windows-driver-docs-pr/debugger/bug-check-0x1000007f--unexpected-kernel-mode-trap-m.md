# <a name="bug-check-0x1000007f-unexpected_kernel_mode_trap_m"></a>Bug 检查0x1000007F：意外\_内核\_模式\_陷阱\_M


意外\_内核\_模式\_陷阱\_M bug 检查的值为0x1000007F。 这表明由 Intel CPU 生成了陷阱，而内核未能捕获此陷阱。

Bug 检查0x1000007F 具有与[**bug 检查 0x7f**](bug-check-0x7f--unexpected-kernel-mode-trap.md)相同的含义和参数（意外\_内核\_模式\_陷阱）。

> [!IMPORTANT]
> 本主题面向程序员。 如果你是在使用计算机时收到蓝屏错误代码的客户，请参阅[排查蓝屏错误](https://www.windows.com/stopcode)。

## <a name="resolution"></a>分辨率
[ **！分析**](https://docs.microsoft.com/windows-hardware/drivers/debugger/-analyze)调试扩展显示有关 bug 检查的信息，可帮助确定根本原因。
