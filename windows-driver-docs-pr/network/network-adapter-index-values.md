# <a name="network-adapter-index-values"></a>网络适配器索引值


中的 HYPER-V 可扩展交换机接口，每个网络适配器，连接到的端口都分配有 NDIS\_切换\_NIC\_索引值。 此索引值标识的可扩展交换机端口上的网络连接。

索引值是唯一的每个网络适配器连接到端口。 尽管大多数网络适配器都需要一个索引值，但可能会与外部网络适配器的端口连接分配多个索引值。 例如，如果外部网络适配器绑定到物理网络适配器的团队，外部网络适配器和每个物理网络适配器分配一个唯一的索引值。

连接到可扩展的交换机中的网络适配器标识通过以下 NDIS\_切换\_NIC\_索引值：

<a href="" id="ndis-switch-default-nic-index"></a>NDIS\_SWITCH\_DEFAULT\_NIC\_INDEX  
此索引值指定为连接到可扩展交换机端口的网络适配器的索引。 此索引值适用于任何直接连接到的可扩展交换机端口，如外部网络适配器、 内部网络适配器或虚拟机 (VM) 网络适配器的网络适配器。

**请注意**  NDIS\_交换机\_默认\_NIC\_定义索引为零。

 

<a href="" id="1-32"></a>1-32  
此索引值指定的基础物理网络适配器的绑定到可扩展交换机外部网络适配器的索引。 索引值分配基于以下配置：

-   如果外部网络适配器绑定到单一物理网络适配器，则会指定的索引值为 1。

-   如果外部网络适配器绑定到的负载平衡故障转移 (LBFO) 团队的物理网络适配器时，整个团队分配的索引值为 1。 LBFO 团队是中的外部网络适配器绑定到的虚拟微型端口缘 LBFO 提供程序的配置。 LBFO 提供程序本身可以绑定到一个或多个物理网络适配器的团队。

    **请注意**  可扩展交换机扩展的基础 LBFO 团队显示为单个网络适配器绑定到外部网络适配器。

     

-   如果外部网络适配器绑定到物理网络适配器的可扩展交换机团队，团队中的每个适配器分配一个唯一索引值，大于或等于 1。 可扩展交换机团队是团队的一个或多个物理网络适配器绑定到外部网络适配器的配置。

有关在其中的物理网络适配器可以绑定到外部网络适配器的不同配置的详细信息，请参阅[的物理网络适配器配置的类型](types-of-physical-network-adapter-configurations.md)。
