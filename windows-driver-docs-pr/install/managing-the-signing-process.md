# <a name="managing-the-signing-process"></a>管理签名过程


签名的过程[驱动程序包](driver-packages.md) [编录文件](catalog-files.md)或驱动程序文件中嵌入签名涉及到以下活动。

### <a name="driver-signing-administration"></a>驱动程序签名管理

这通常由发布者的项目管理和软件发布服务，并包括以下主题：

-   [选择适当的签名类型](selecting-the-appropriate-signature-type.md)

-   获取所需[发布证书](release-certificates.md)或[测试证书](test-certificates.md)

-   [管理数字签名或代码签名密钥](managing-the-digital-signature-or-code-signing-keys.md)

### <a name="driver-development"></a>驱动程序开发

这通常由发布者的开发团队处理，包括以下方案：

-   实现该驱动程序。

-   创建进行内部测试签名的驱动程序包。 有关测试签名的信息，请参阅[签名驱动程序开发和测试期间](signing-drivers-during-development-and-test.md)。

-   创建发布签署的驱动程序包。 有关如何进行签名版本的驱动程序的信息，请参阅[签名的驱动程序公共版本](signing-drivers-for-public-release.md)。
